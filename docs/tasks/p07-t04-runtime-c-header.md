# p07-t04: Unified C Runtime Header

## Purpose
Create the unified C runtime header `runtime/ax_runtime.h`. Every `.c` file generated by the AXIOM C-Backend begins with `#include "ax_runtime.h"`. This single include provides all built-in type definitions, allocator access, generational reference functions, panic handling, and helper macros needed by generated code.

## Context
The C-Backend (Phase 08) generates C11 source files. Rather than emitting type definitions and runtime declarations inline into every generated file, the backend emits a single `#include "ax_runtime.h"` and relies on this header to provide everything. This separation keeps generated code clean, makes the runtime independently updatable, and ensures consistency across all generated files.

The header must be self-contained: including it in a fresh `.c` file with no other includes must compile without errors. It must also be idempotent (safe to include multiple times via include guards).

## Inputs
- `runtime/axalloc/axalloc.h` — allocator and AxHeader/AxRef types (p07-t01)
- `runtime/axalloc/genref.h` — ax_deref, ax_make_ref (p07-t02)
- `runtime/panic/panic.h` — ax_panic, ax_bounds_check, ax_assert (p07-t03)
- C standard library headers: `<stdint.h>`, `<stdbool.h>`, `<stddef.h>`, `<string.h>`

## Outputs
- `runtime/ax_runtime.h` — the master include file
- `runtime/test_runtime_header.c` — smoke test that the header compiles in isolation

## Dependencies
- p07-t02 (genref.h, which depends on axalloc.h from p07-t01)
- p07-t03 (panic.h)

## Subsystems Affected
- C-Backend (primary consumer — emits this include in every generated file)
- All runtime subsystems (all public APIs consolidated here)
- Tests (any C test that needs runtime functions includes this)

## Detailed Requirements

### C Type Mappings (AXIOM → C)
All AXIOM primitive types must have stable C typedef aliases:
```c
// Signed integers
typedef int8_t   ax_i8;
typedef int16_t  ax_i16;
typedef int32_t  ax_i32;
typedef int64_t  ax_i64;

// Unsigned integers
typedef uint8_t  ax_u8;
typedef uint16_t ax_u16;
typedef uint32_t ax_u32;
typedef uint64_t ax_u64;

// Floating point
typedef float    ax_f32;
typedef double   ax_f64;

// Boolean
typedef _Bool    ax_bool;
#define AX_TRUE  ((ax_bool)1)
#define AX_FALSE ((ax_bool)0)

// Void (for completeness; used in pointer types)
typedef void     ax_void;
```

### String Type
AXIOM `string` is an immutable length-prefixed byte slice. The C representation:
```c
typedef struct {
    const ax_u8* ptr;  // pointer to UTF-8 bytes (not null-terminated)
    ax_u64       len;  // byte length
} ax_string;

// Convenience macro for string literals
#define AX_STR(literal) \
    ((ax_string){ .ptr = (const ax_u8*)(literal), .len = sizeof(literal) - 1 })
```

### Slice Type Template
AXIOM `[T]` is a growable slice. The C-Backend generates a concrete struct for each element type. Provide a macro to declare slice types:
```c
// Usage: AX_SLICE_TYPE(ax_i32) expands to the struct definition name ax_slice_i32
#define AX_SLICE_DECL(T, name) \
    typedef struct { T* ptr; ax_u64 len; ax_u64 cap; } name

// Pre-declare common slices
AX_SLICE_DECL(ax_i8,  ax_slice_i8);
AX_SLICE_DECL(ax_i32, ax_slice_i32);
AX_SLICE_DECL(ax_i64, ax_slice_i64);
AX_SLICE_DECL(ax_u8,  ax_slice_u8);   // byte slice (used for string builder etc.)
AX_SLICE_DECL(ax_f32, ax_slice_f32);
AX_SLICE_DECL(ax_f64, ax_slice_f64);
```

### Utility Macros
```c
// Silence unused variable warnings (common in generated code for loop vars)
#define AX_UNUSED(x) ((void)(x))

// Likely / unlikely branch hints
#define AX_LIKELY(x)   __builtin_expect(!!(x), 1)
#define AX_UNLIKELY(x) __builtin_expect(!!(x), 0)

// Static array length
#define AX_ARRAY_LEN(arr) (sizeof(arr) / sizeof((arr)[0]))

// Min / Max without UB (works for same-type arithmetic types)
#define AX_MIN(a, b) ((a) < (b) ? (a) : (b))
#define AX_MAX(a, b) ((a) > (b) ? (a) : (b))

// Compile-time assertion
#define AX_STATIC_ASSERT(cond, msg) _Static_assert(cond, msg)
```

### Entry Point Wrapper
Generated `main.c` files define `ax_main()` (the AXIOM entry point) and the C-Backend emits a wrapper `main()`:
```c
// Declared in ax_runtime.h; the wrapper calls ax_set_program_name then ax_main
extern int ax_main(int argc, char** argv);

static int main(int argc, char** argv) {
    ax_set_program_name(argv[0]);
    return ax_main(argc, argv);
}
```

This wrapper must be emitted only in the file containing the `main` function, controlled by a `#define AX_EMIT_MAIN` guard that the C-Backend sets in the top-level file.

### Include Order in `ax_runtime.h`
The header must include its dependencies in this order to avoid forward-declaration issues:
1. System headers: `<stdint.h>`, `<stdbool.h>`, `<stddef.h>`, `<stdio.h>`, `<string.h>`, `<stdlib.h>`
2. Type definitions (the `ax_` typedefs)
3. `"axalloc/axalloc.h"` (AxHeader, AxRef, ax_alloc, ax_free, ax_realloc, ax_alloc_size)
4. `"axalloc/genref.h"` (ax_deref, ax_make_ref, ax_invalidate, ax_ref_valid, AX_NULL_REF)
5. `"panic/panic.h"` (ax_panic, ax_bounds_check, ax_assert)
6. Utility macros
7. Slice type declarations
8. Conditional entry point wrapper

### Path Resolution
The C-Backend places generated `.c` files in a temp directory alongside a copy of (or symlink to) `ax_runtime.h` and its dependencies. The `#include "ax_runtime.h"` in generated code uses a relative path, so the runtime headers must be co-located or passed via `-I runtime/` to the GCC invocation.

The `axc build` pipeline (p08-t09) will:
1. Copy `runtime/ax_runtime.h` and subdirectories to a temp staging area
2. Invoke GCC with `-I <staging_area>` so `#include "ax_runtime.h"` resolves

## Implementation Steps

### Step 1: Create `runtime/ax_runtime.h`
```c
#pragma once

// ---- System headers ----
#include <stdint.h>
#include <stdbool.h>
#include <stddef.h>
#include <stdio.h>
#include <string.h>
#include <stdlib.h>

// ---- Primitive type aliases ----
typedef int8_t   ax_i8;
typedef int16_t  ax_i16;
typedef int32_t  ax_i32;
typedef int64_t  ax_i64;
typedef uint8_t  ax_u8;
typedef uint16_t ax_u16;
typedef uint32_t ax_u32;
typedef uint64_t ax_u64;
typedef float    ax_f32;
typedef double   ax_f64;
typedef _Bool    ax_bool;
#define AX_TRUE  ((ax_bool)1)
#define AX_FALSE ((ax_bool)0)

// ---- String type ----
typedef struct {
    const ax_u8* ptr;
    ax_u64       len;
} ax_string;
#define AX_STR(lit) ((ax_string){ .ptr=(const ax_u8*)(lit), .len=sizeof(lit)-1 })

// ---- Runtime subsystems ----
#include "axalloc/axalloc.h"
#include "axalloc/genref.h"
#include "panic/panic.h"

// ---- Slice types ----
#define AX_SLICE_DECL(T, name) \
    typedef struct { T* ptr; ax_u64 len; ax_u64 cap; } name
AX_SLICE_DECL(ax_i8,  ax_slice_i8);
AX_SLICE_DECL(ax_i16, ax_slice_i16);
AX_SLICE_DECL(ax_i32, ax_slice_i32);
AX_SLICE_DECL(ax_i64, ax_slice_i64);
AX_SLICE_DECL(ax_u8,  ax_slice_u8);
AX_SLICE_DECL(ax_u16, ax_slice_u16);
AX_SLICE_DECL(ax_u32, ax_slice_u32);
AX_SLICE_DECL(ax_u64, ax_slice_u64);
AX_SLICE_DECL(ax_f32, ax_slice_f32);
AX_SLICE_DECL(ax_f64, ax_slice_f64);

// ---- Utility macros ----
#define AX_UNUSED(x)       ((void)(x))
#define AX_LIKELY(x)       __builtin_expect(!!(x), 1)
#define AX_UNLIKELY(x)     __builtin_expect(!!(x), 0)
#define AX_ARRAY_LEN(arr)  (sizeof(arr) / sizeof((arr)[0]))
#define AX_MIN(a,b)        ((a) < (b) ? (a) : (b))
#define AX_MAX(a,b)        ((a) > (b) ? (a) : (b))
#define AX_STATIC_ASSERT(cond, msg) _Static_assert(cond, msg)

// ---- Entry point wrapper (only in main compilation unit) ----
#ifdef AX_EMIT_MAIN
extern int ax_main(int argc, char** argv);
int main(int argc, char** argv) {
    ax_set_program_name(argc > 0 ? argv[0] : "<axiom>");
    return ax_main(argc, argv);
}
#endif
```

### Step 2: Create `runtime/test_runtime_header.c`
```c
// Smoke test: including ax_runtime.h in isolation must compile cleanly.
#include "ax_runtime.h"

// Verify type sizes
AX_STATIC_ASSERT(sizeof(ax_i8)  == 1, "ax_i8 size");
AX_STATIC_ASSERT(sizeof(ax_i16) == 2, "ax_i16 size");
AX_STATIC_ASSERT(sizeof(ax_i32) == 4, "ax_i32 size");
AX_STATIC_ASSERT(sizeof(ax_i64) == 8, "ax_i64 size");
AX_STATIC_ASSERT(sizeof(ax_f32) == 4, "ax_f32 size");
AX_STATIC_ASSERT(sizeof(ax_f64) == 8, "ax_f64 size");
AX_STATIC_ASSERT(sizeof(ax_string) == 16, "ax_string size");

int main(void) {
    // Exercise each type and macro
    ax_i32 x = 42;
    ax_string s = AX_STR("hello");
    ax_bool b = AX_TRUE;
    AX_UNUSED(x); AX_UNUSED(s); AX_UNUSED(b);

    // Allocator smoke test
    void* p = ax_alloc(64);
    AxRef ref = ax_make_ref(p);
    void* p2 = ax_deref(ref);
    AX_UNUSED(p2);
    ax_free(p);

    // Bounds check
    ax_bounds_check(0, 10);
    ax_assert(1, "should not panic");

    return 0;
}
```

### Step 3: Update `runtime/axalloc/Makefile` (or create `runtime/Makefile`)
```makefile
CC     = gcc
CFLAGS = -O2 -Wall -Wextra -Werror -std=c11 -I.

PANIC_STUB = -Dpanic_stub
# For smoke test, provide ax_panic as a simple abort wrapper
SMOKE_OBJS = axalloc/axalloc.o panic/panic.o

all: test_runtime_header

axalloc/axalloc.o:
	$(MAKE) -C axalloc axalloc.o

panic/panic.o:
	$(MAKE) -C panic panic.o

test_runtime_header: test_runtime_header.c $(SMOKE_OBJS)
	$(CC) $(CFLAGS) -rdynamic test_runtime_header.c $(SMOKE_OBJS) -o test_runtime_header
	./test_runtime_header

clean:
	rm -f test_runtime_header
	$(MAKE) -C axalloc clean
	$(MAKE) -C panic clean
```

## Test Plan
1. `#include "ax_runtime.h"` in a fresh `.c` file compiles without warnings (`-Wall -Wextra -Werror`)
2. `sizeof(ax_i8)` == 1, ..., `sizeof(ax_f64)` == 8 (static asserts)
3. `sizeof(ax_string)` == 16 (8-byte ptr + 8-byte len on 64-bit)
4. `AX_STR("hello").len` == 5 (compile-time)
5. `ax_alloc` → `ax_make_ref` → `ax_deref` round trip works
6. `ax_bounds_check` and `ax_assert` are callable
7. Including the header twice (double-include) does not produce errors
8. `AX_EMIT_MAIN` guard: defining it before include causes `main()` to be emitted

## Validation Checklist

## Risks & Mitigations
- **Risk**: `__builtin_expect` is GCC/Clang-specific; MSVC does not support it. **Mitigation**: Guard with `#ifndef _MSC_VER` and provide no-op fallback macros for MSVC.
- **Risk**: Relative include paths (`"axalloc/axalloc.h"`) may break when the header is installed to a different directory. **Mitigation**: The `axc build` pipeline always copies the full `runtime/` tree to the staging area, preserving the directory structure.
- **Risk**: `AX_EMIT_MAIN` guard causes `main()` to be a static function emitted in a header, which is unusual. **Mitigation**: The guard is only meant to be defined in exactly one `.c` file per binary. Document this clearly.
