# p16-t09: std.json — JSON Serialization

## Purpose
Implement JSON serialization and deserialization for AXIOM types using compile-time reflection to auto-derive encode/decode for structs and sum types.

## Context
JSON is the universal interchange format. AXIOM's `std.json` uses the type system to derive serializers at compile time — no runtime reflection needed. Structs annotated with `#[json]` get `to_json()` and `from_json()` methods generated by the compiler.

## Inputs
- TypedAST from p04 — struct field names and types
- `#[json]` attribute on types
- `std.string` for string manipulation
- `std.collections` for Array/HashMap

## Outputs
- `stdlib/json/json.ax` — JsonValue type + parser + serializer
- `stdlib/json/derive.ax` — compile-time codegen for #[json]
- Compiler extension: `#[json]` attribute handler in codegen

## Dependencies
- p05-t01: generic-type-representation — generic JsonValue type
- p16-t02: std-string — string manipulation for JSON output
- p04: type checker — processes `#[json]` attribute

## Detailed Requirements

```axiom
# stdlib/json/json.ax

type JsonValue:
    Null
    Bool(bool)
    Number(f64)
    Str(str)
    Array(Array[JsonValue])
    Object(HashMap[str, JsonValue])

fn parse(s: str) -> Result[JsonValue, JsonError]
fn stringify(val: JsonValue) -> str
fn stringify_pretty(val: JsonValue, indent: u32) -> str

type JsonError:
    UnexpectedChar(char: u32, pos: u32)
    UnexpectedEnd
    InvalidUtf8
    NumberOverflow

# Derive interface
interface JsonEncode:
    fn to_json(self) -> JsonValue
interface JsonDecode:
    fn from_json(val: JsonValue) -> Result[Self, JsonError]
```

`#[json]` auto-derive example:
```axiom
#[json]
type Person:
    var name: str
    var age:  i32
    var email: Option[str]

# Generated:
# impl JsonEncode for Person:
#     fn to_json(self) -> JsonValue:
#         JsonValue::Object({
#             "name": JsonValue::Str(self.name),
#             "age":  JsonValue::Number(f64(self.age)),
#             "email": match self.email { Some(v) -> JsonValue::Str(v), None -> JsonValue::Null }
#         })
# impl JsonDecode for Person:
#     fn from_json(val: JsonValue) -> Result[Person, JsonError]: ...
```

Parser: recursive descent, handles all JSON grammar.

Number parsing: parse as f64, detect integer range for lossless i64 storage.

## Implementation Steps

1. Create `stdlib/json/json.ax` — JsonValue type + manual parse/stringify.
2. Implement recursive descent JSON parser.
3. Implement `stringify()` / `stringify_pretty()`.
4. Create `stdlib/json/derive.ax` — JsonEncode/JsonDecode interfaces.
5. Implement `#[json]` compiler attribute handler generating impl blocks.
6. Write tests parsing and round-tripping JSON.

## Test Plan
- `TestParseNull`: `null` → JsonValue::Null
- `TestParseObject`: `{"k": 1}` → Object with one entry
- `TestParseArray`: `[1,2,3]` → Array of Numbers
- `TestParseString`: escape sequences `\n`, `A` handled
- `TestStringify`: JsonValue → string → parse → equal
- `TestDeriveEncode`: `#[json] Person` → `to_json()` produces correct Object

## Risks & Mitigations
| Risk | Mitigation |
|------|-----------|
| Number precision (f64 can't represent all i64) | Detect integer-range numbers, store as i64 variant in extended mode |
| Circular references in to_json | AXIOM ownership prevents circular structures; no issue |
