
### PHẦN 1: CẤU TRÚC KHO LƯU TRỮ (REPOSITORY STRUCTURE)

Kho lưu trữ sử dụng kiến trúc Monorepo, chứa toàn bộ chuỗi công cụ (toolchain) của Axiom trong một dự án duy nhất.

```text
axiom-lang/
├── cmd/
│   └── axc/                 # Entry point cho compiler CLI (build, run, fmt, lsp)
├── compiler/
│   ├── ast/                 # Định nghĩa Flat AST (Array-based AST)
│   ├── lexer/               # Zero-copy Lexer
│   ├── parser/              # Recursive Descent + Pratt Parser
│   ├── sema/                # Semantic Checker & Lazy Field Analysis
│   └── types/               # Type System (Primitives, Type Inference)
├── ir/
│   ├── air/                 # Axiom IR definitions (SSA form)
│   ├── builder/             # AST to AIR lowerer
│   └── opt/                 # Optimization Passes (CTGC, DCE, Inlining)
├── codegen/
│   ├── cgen/                # C-Backend (Transpiler for MVP)
│   └── native/              # Native x86-64 Assembler (Postponed/V2)
├── runtime/                 # Viết bằng C/Axiom để nhúng vào binary
│   ├── axalloc/             # Actor-local memory allocator (mimalloc-inspired)
│   ├── actors/              # Lightweight processes & channel
│   └── panic/               # Panic handler & Stacktrace
├── tools/
│   ├── lsp/                 # Language Server Protocol & .axmeta emitter
│   └── pkg/                 # Trình quản lý gói nội bộ (Axiom Package Manager)
├── std/                     # Thư viện chuẩn viết bằng Axiom (.ax)
├── tests/                   # Bộ Fuzzing và Spec tests
└── go.mod                   # (Nếu dùng Go làm Bootstrap compiler)
```

### PHẦN 2: THIẾT KẾ CÁC MODULE (MODULES BREAKDOWN)

#### 1. Compiler Modules
*   **`lexer`**: Quét luồng mã nguồn. Cấu trúc Token là `struct { Kind uint8, Pos uint32, Len uint16 }`. Phải xử lý được Indentation (thu thập khoảng trắng đầu dòng thành `Indent` hoặc `Dedent` token).
*   **`ast`**: Cây cú pháp phẳng. Dùng một mảng `Nodes []Node`. Liên kết cha-con thông qua `ChildIdx int32` và `SiblingIdx int32`.
*   **`parser`**: Sử dụng luồng đệ quy xuống (Recursive Descent) cho `Statements` và Pratt Parser cho `Expressions`. Cập nhật mảng phẳng AST trực tiếp.
*   **`sema`**: Khởi tạo Bảng ký hiệu tĩnh (Symbol Table). Implement luồng *Lazy Field Analysis* (chỉ duyệt những hàm được gọi từ `main`).

#### 2. IR Modules (AIR)
*   **`air`**: Định nghĩa tập lệnh 3 địa chỉ (3-address code). VD: `OpIAdd, OpLoad, OpStore, OpCall, OpAlloc`.
*   **`builder`**: Nhận AST đã gắn kiểu (Typed AST) và chuyển thành luồng lệnh SSA.
*   **`opt`**: Module tối ưu hóa. Mức MVP chỉ cần implement **Escape Analysis** (quyết định Heap hay Stack) và **CTGC** (chèn `=destroy` ngầm định ở cuối scope).

#### 3. Runtime Modules
*   **`axalloc`**: Bộ cấp phát siêu nhẹ. Mức MVP: Gọi `malloc()` của C nhưng cấp thêm 8-byte Header để lưu `GenerationID`.
*   **`actors`**: Mức MVP: Bọc (Wrap) `pthread` của C để chạy đồng thời. M:N Scheduler sẽ làm sau.

#### 4. Codegen & Linker
*   **`cgen`**: Chuyển đổi AIR hoặc Typed AST trực tiếp thành mã chuẩn C11. 
*   **`linker`**: Ở MVP, module này chỉ tạo subprocess gọi `gcc -O3 output.c -o app` và bắt kết quả.

### PHẦN 3: SƠ ĐỒ PHỤ THUỘC & ĐƯỜNG GĂNG (DEPENDENCY GRAPH & CRITICAL PATH)

**Dependency Graph:**
`EBNF Grammar` $\rightarrow$ `Lexer` $\rightarrow$ `AST Structs` $\rightarrow$ `Parser` $\rightarrow$ `Symbol Table` $\rightarrow$ `Type Checker` $\rightarrow$ `C-Backend` $\rightarrow$ `Standard Library`.

**CRITICAL PATH (Đường Găng - Không được phép tắc nghẽn):**
1. Phải hoàn thành **AST Structs (Flat Array)** đầu tiên. Mọi thành phần khác đều phụ thuộc vào nó.
2. **C-Backend Codegen** phải được viết song song ngay sau khi có Parser. Bạn cần nhìn thấy đầu ra C để chứng minh AST hợp lệ.
3. Không làm IR (AIR) và Tối ưu hóa (Opt) ở tháng đầu tiên. Đi thẳng từ `AST` $\rightarrow$ `C-Backend` để ra sản phẩm nhanh nhất.

### PHẦN 4: ROADMAP THỰC THI CHO 1 NGƯỜI (MILESTONES & WEEKLY GOALS)

Đây là kế hoạch hành động chớp nhoáng trong **3 tháng đầu tiên (12 tuần)** để bạn đạt được "Axiom v0.0.1". 

#### Milestone 1: Nền tảng Phân tích Cú pháp (Tuần 1 - Tuần 4)
*Mục tiêu: Đọc được file `.ax` và in ra được cây cấu trúc JSON.*
*   **Tuần 1:** Viết tệp `GRAMMAR.ebnf`. Setup Go Repository. Dùng AI sinh `lexer`. Viết Test cho Lexer (kiểm tra Indent/Dedent).
*   **Tuần 2:** Định nghĩa cấu trúc mảng `ast.Node`. Bắt đầu viết `parser` (Statements: `fn`, `struct`, `if`, `var`).
*   **Tuần 3:** Cài đặt Pratt Parser cho Expressions (Toán tử `+ - * /`, Gọi hàm).
*   **Tuần 4:** Viết một bộ in AST (AST Printer). Truyền 1 file `.ax` vào và yêu cầu compiler in ra toàn bộ cây AST.

#### Milestone 2: Ngữ nghĩa & Transpiler thô (Tuần 5 - Tuần 8)
*Mục tiêu: Biên dịch được chương trình "Hello World" và tính Fibonacci ra C.*
*   **Tuần 5:** Xây dựng `SymbolTable` và `Scope`. Phân giải tên biến (VD: báo lỗi nếu biến chưa khai báo).
*   **Tuần 6:** Viết `TypeChecker` cơ bản (`int`, `bool`, `string`). Chặn gán sai kiểu.
*   **Tuần 7:** Bắt đầu module `cgen`. Viết bộ ánh xạ AST Node sang chuỗi mã C (VD: `NodeIf` $\rightarrow$ `if (...) { ... }`).
*   **Tuần 8:** Hoàn thiện `cgen`. Tích hợp lệnh `axc build main.ax` gọi ngầm GCC. Thành công chạy file `.exe` đầu tiên.

#### Milestone 3: Quản lý Bộ nhớ & Ownership (Tuần 9 - Tuần 12)
*Mục tiêu: Hiện thực hóa điểm độc nhất của Axiom - An toàn bộ nhớ không cần GC.*
*   **Tuần 9:** Hiện thực quy tắc `Single Ownership` vào Type Checker. Chặn việc sao chép object (chỉ cho phép Move hoặc Borrow).
*   **Tuần 10:** Tự động chèn (Inject) các node `free()` hoặc `=destroy()` vào AST khi một biến `Owner` thoát khỏi Tầm vực (Scope).
*   **Tuần 11:** Implement `Generational References` (Runtime). Sửa C-Backend để thêm `gen_id` vào struct C, sinh mã check ID mỗi khi giải tham chiếu con trỏ.
*   **Tuần 12:** Fix bugs. Dùng AI viết các đoạn mã test phức tạp. Công bố Axiom v0.0.1.

#### Các bước tiếp theo sau Tuần 12:
*   Tuần 13-16: Bắt đầu viết Module Thư viện chuẩn `std.fs`, `std.net` bằng file `.ax`.
*   Tuần 17-20: Tích hợp Hệ thống Macro nội tuyến `#run`.
*   Tuần 21-24: Xây dựng bộ xuất metadata cho AI (`.axmeta`) và Language Server (LSP).
